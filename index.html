    The Path To Security Automation Enlightenment

    Infrastructure As Code

    • Manage infrastucture using a descriptive model
    • Generates same environment every time it is applied
    • Created to solve "environmental drift" in release pipelines
    • Idempotence a key feature

    Infrastructure As Code

    
resource "google_sql_database instance" "master" {
  name	           = "master-instance"
  database_version = "POSTGRES_11"
  region	   = "us-centra1"
								
  settings {
    tier = "db-f1-micro"
  }
}

    But can we do better?

    Infrastructure As Data

    • Manage infrastructure using a declarative model
    • Treats infrastructure as data, more portable than code
    • Data can be imported/modified by other tools
    • Language agnostic, version controlled

    Infrastructure As Data

    
apiVersion: sql.cnrm.cloud.google.com/v1beta1
  kind: SQLInstance
  metadata:
    name: sqlinstance-sample-mysql
  spec:
    databaseVersion: MYSQL_5_7
    region: us-central1
  settings:
    tier: db-g1-small
    diskSize: 25
    diskType: PD_SSD
    availabilityType: REGIONAL
    backupConfiguration:
    binaryLogEnabled: true
    enabled: true
